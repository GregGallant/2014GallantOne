All Tomorrow's Tech

There's been quite an influx of new technologies when it comes to front-end development. Within various companies the biggest challenge has been the maintainability of code and the presentation of organized data. The creation of states and stores was usually handled on the backend using ideas provided by a framework or an ORM. ReactJS and the component driven virtual DOM along with Redux's immutable state store ideology has, in my opinion, proven itself to be quite effective at producing quality responses. I'm currently migrating a lot of jQuery scripts into reactJS driven components, translating some of the jQuery calls back into vanilla javascript to assist with speed and time.

Previously I created a few CRUD setups using AngularJS, and while powerful, there was a large disconnect between structure and setup; there was too many ways to make different calls to different controllers, service managers, providers and so on. For one developer, this could be considered flexible. For a team, this would turn into a desert combing bughunt. AngularJS was an idealistic framework, ReactJS was a library and that ultimately was the deciding factor into what technology to personally adopt.
